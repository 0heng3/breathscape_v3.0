# BreathScape Demo 实现计划

## 1. 目标

依据 `BreathScape Demo 功能说明.pdf`，实现一个适合 iPad 展示的 Web App 原型，完整呈现以下核心链路：

心情进入 -> AI 场景引导 -> 声景线描创作 -> 生成式反馈 -> 呼吸收尾 -> 情绪日记保存。

本版本不做心理诊断、不做量表、不做真实面部情绪判断、不做父母端完整功能。AI 能力以规则原型模拟，确保演示重点聚焦在非评判式引导、线描创作、声音反馈和日记回看。

## 2. 技术方案

- 前端框架：Vite + React。
- 视觉实现：CSS 纸张肌理、水彩色块、DOM 动画、Canvas 绘制层。
- 图标：lucide-react。
- 声音反馈：Web Audio API 合成轻风、雨滴、草叶沙沙、开花泛音、阳光铃音，避免外部音频依赖。
- 数据保存：localStorage 保存日记卡，支持本地演示回看。
- 适配目标：iPad 横屏优先，兼容竖屏和桌面浏览器；触控笔/手指均可绘制。

## 2.1 竞赛优秀作品调研与设计迁移

本轮更新参考了“中国高校计算机大赛-移动应用创新赛”公开报道与同类儿童创作/情绪产品的展示方式：

- 第八届移动应用创新赛公开报道显示，赛事作品覆盖健康、传统文化、乡村振兴、环境等真实议题，强调创意赛道、跨学科和面向应用的完整体验。该方向提示 BreathScape 不应只是功能列表，而要呈现清晰的“情绪场景日记系统”叙事。
- 上海交通大学《汉字修复师》获得第八届最高奖“最具创新奖”。公开介绍中，它把汉字文化转化为“寻找字根并拼合成字”的解谜动作。可迁移到 BreathScape 的思路是：把抽象情绪转成可操作的“线条元素选择、绘制、生成反馈”。
- 同届《烟消云散》等作品体现了移动应用创新赛常见的游戏化表达：通过场景变化反馈用户行为，而不是只展示表单或文字说明。BreathScape 因此强化花园状态变化、工具计数、故事卡和本周小地图。
- Red Dot 上的《Chinese Character Repairer - Word Searching Game》页面保留了多张实际界面图，可见其不是单一插画，而是通过地图式场景、关卡路径、纸张质感卡片、角色引导和局部交互点构成完整体验。这一方向迁移到 BreathScape：花园不再只是一个发光土坡，而要像一个可探索的情绪地图，有远景、河流、桥、门、灯、路径和状态标记。
- 2024 年移动应用创新赛报道中的《绘游戏》是与本项目最接近的儿童绘画方向：它面向儿童绘画启蒙，上传画作后提取元素，生成立体场景和小故事，并让静态图画动起来。BreathScape 的对应实现是：儿童线描一笔后，场景生态层立即发生变化，日记故事同步生成，但仍保持非诊断式表达。
- 2023 年报道中的《声达》通过声音和动画辅助训练，《化学元素 AR 探索》通过空间/AR 让抽象知识更可操作；对应到本 Demo，是把“情绪调节”隐藏在画线、听声音、看花园生长的交互过程中，避免测评感。
- 同类儿童 AI 绘画产品如 Droodle 强调 iPad 画布、手指/触控笔涂鸦和“让作品动起来”的即时魔法反馈；儿童日记产品如 Angel Diary 强调无压力、无标签、无诊断。这两点分别对应本 Demo 的画布反馈和非评判式文案边界。

## 2.2 儿童 App / App Store 页面风格补充分析

继续检索 App Store 与儿童产品官网后，提炼出以下可迁移规律：

- Pok Pok：App Store 页面明确强调 low-stimulation、open-ended play、no rules/no levels/no winning，并强调 slow-paced animations、soft hand-recorded sounds。其界面展示重点不是复杂菜单，而是完整可探索玩具场景。迁移原则：BreathScape 要减少说明型 UI，突出“孩子直接点、画、听、看变化”的开放式探索。
- Sago Mini World：官网以大幅角色/场景图和“Something to discover for every preschooler”组织内容，列表不是冷冰冰的功能表，而是 Airport、Apartment、Big City、Boats 等可进入的小世界。迁移原则：BreathScape 的花园需要更像一个可进入的小世界，而不是一张平面卡片。
- Moshi Kids：App Store 描述强调 safe、calming、ad-free、kid-safe，内容围绕睡眠、放松、呼吸、故事和声音。迁移原则：BreathScape 的动画速度、颜色对比、音效和文案要有“安抚感”，不能为了好看变成高刺激游戏。
- Headspace Kids：儿童 mindfulness 页面把“deep breath、imagination、explore every feeling”结合起来，并按家庭内容卡片组织。迁移原则：BreathScape 的呼吸收尾要保留，但入口和画布要用想象场景承接，而不是像训练工具。
- Toca Boca / Sago Mini 一类儿童应用常见做法：第一屏即有强视觉资产；页面组件大、圆、少文字；按钮颜色明确；儿童不需要阅读复杂说明；交互反馈来自场景本身而不是弹窗。

落到本 Demo 的设计规则：

- 布局：iPad 横屏采用“左/中大场景 + 右侧轻提示”的舞台布局；首页保留大幅插画，文案不压在画面上；创作页画布占主视觉 55%-65%。
- 色彩：以黄紫为主，但避免全屏高饱和紫。黄色负责安全、温暖和行动；紫色负责幻想、夜光和情绪空间；薄荷绿、湖蓝、花粉只用于状态反馈。
- 字体：标题保持粗圆、字重高；正文行高宽松；儿童操作标签保持短词；避免小字号密集说明。
- 元素：使用高完成度插画资产作为底图，再叠加可交互状态层。避免只靠 CSS 圆形/椭圆堆出“玩具感”。
- 按钮：主按钮使用高识别渐变和投影；工具按钮大于 64px，图标颜色跟当前元素一致，选中态必须一眼可见。
- 交互：每一笔至少触发三件事：画布留下线条、场景状态有变化、右侧反馈文案更新。高级反馈再加入声音和日记故事。
- 页面大小：以 1024x768 iPad 横屏为主要验收尺寸；画布必须保持 4:3 附近的儿童绘画比例，底部提示不能遮住主要绘制区。

## 2.3 当前页面诊断与改造方向

当前版本相比最初已经有黄紫主题和地图元素，但仍有两个问题：

- 背景主要由 CSS 几何形状组成，远看有层次，近看仍像临时原型，缺少 App Store 截图级别的完成度。
- 花园空间存在但缺少自然材质、光影和真实插画边缘，儿童产品常见的“可进入故事世界”感觉不足。

本轮改造方向：

- 生成一张项目专用的情绪花园位图插画，作为首页与画布底图。
- HTML/CSS 不再承担全部视觉表现，而改为承担 UI、交互状态、笔触、计数和反馈层。
- 保留地图化方向，但让地图细节成为插画的一部分，避免重复堆叠造成玩具感。

## 3. 页面与功能拆解

### 3.1 进入页

- 呈现安静的空白花园、一颗种子和温和引导文案。
- 提供“开始今日花园”按钮。
- 设计重点：降低表达压力，明确这里不是考试、测评或绘画比赛。

### 3.2 今日心情选择

- 使用天气/场景隐喻选择今日状态：
  晴天、小雨、大风、小火苗、雾蒙蒙、空土地、我也不知道。
- 底部显示“没关系，心情不一定马上说得清。我们可以先一起画出来。”
- 选择后进入 AI 场景初始化。

### 3.3 AI 场景初始化

- 根据心情隐喻显示非诊断式场景文案和推荐元素。
- 不出现“焦虑”“难过”“冷静”等心理判断。
- 提供进入画布的按钮。

### 3.4 场景共创画布

- 提供 5 个核心元素：风、雨、草、花、阳光。
- 通过 Canvas 记录儿童线描轨迹。
- 每次完成一笔后：
  - 根据当前工具与线条速度更新场景状态。
  - 播放对应合成音效。
  - 显示温和反馈文案。
  - 触发视觉动画：风线、雨滴、小草、花朵、阳光亮度等。
- AI 引导采用规则模拟：
  - 快速/大量画风时邀请加入云、草等放缓元素。
  - 雨较多时提示土地柔软、小草可能冒出。
  - 画面较空时允许继续安静或加入声音。

### 3.5 呼吸收尾

- 展示缓慢放大/缩小光点。
- 引导 3 次简单呼吸。
- 同步弱化风雨动效，使画面安放下来。

### 3.6 今日花园日记卡

- 根据添加元素顺序生成温和故事回顾。
- 可给作品命名。
- 保存到 localStorage。
- 不出现疗效判断或风险评分。

### 3.7 我的情绪场景日记

- 展示今日花园、历史花园和本周小地图。
- 提供基于本地日记元素统计的温和周回顾。

## 4. 视觉与交互标准

- 主色调更新为“晨光黄 + 花影紫”的高识别度方案，辅以薄荷绿、湖蓝和花瓣粉，避免单一紫色或暗沉纸张感。
- 主场景从“空白纸面/玩具土坡”升级为“情绪花园地图”：有远山、紫色晨雾、金色光路、河带、小桥、月门、聆听小屋、灯点、路径、地图标记和可生长的自然元素。
- 入口页需要第一屏即呈现品牌气质和应用主题，不只是淡色背景加文字；按钮要有明确主行动层级。
- 创作页参考竞赛作品的游戏化表达，强调“我画一笔，场景马上回应”：工具选中态、元素计数、动画、场景地图标记和反馈文案都要更鲜明。
- 字体：系统圆润无衬线，较大字号，适合儿童和 iPad 触控阅读。
- 按钮与工具：使用图标+短文案，触控目标不小于 44px。
- 画布：横屏 iPad 下保持稳定大画面，工具栏不遮挡绘制区。
- 动效：缓慢、轻柔、可预测，不制造强闪烁。

## 5. 验收清单

- 能从进入页完成完整链路并保存日记。
- 7 个心情选项都能生成不同初始化文案与推荐元素。
- 5 个绘制元素都有视觉反馈、文案反馈和声音反馈。
- 日记列表能展示保存结果与本周回顾。
- 构建命令 `npm.cmd run build` 成功。
- 使用浏览器以 iPad 视口预览，确认核心内容不重叠，画布可操作。
