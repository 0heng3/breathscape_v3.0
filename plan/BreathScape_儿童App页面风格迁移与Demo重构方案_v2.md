# BreathScape「息境」儿童 App 页面风格迁移与 Demo 重构方案 v2

> 文档定位：本方案用于指导 BreathScape Demo 的新一轮视觉、交互与前端实现重构。它不是单纯的竞品调研汇总，而是把儿童 App / App Store 页面风格规律转译为可执行的页面规范、组件规范、AI 规则原型、技术实现与验收标准。

---

## 0. 版本摘要

项目名称：息境 BreathScape  
副标题：基于声景线描的儿童情绪场景日记原型  
目标用户：6–12 岁儿童，兼顾课堂展示、家庭亲子共创、心理健康教育与艺术疗愈课程中的轻度情绪表达场景  
核心链路：心情进入 → AI 场景引导 → 声景线描创作 → 生成式反馈 → 呼吸收尾 → 情绪日记保存  
当前版本目标：完成一版适合 iPad 横屏演示的 Web App 原型，视觉达到儿童 App Store 截图级别，交互达到“可演示、可理解、可感知反馈”的程度。

本版本的重构重点有五个：

1. 从“功能展示型 Demo”转为“可进入的儿童情绪小世界”。
2. 从“页面说明文字驱动”转为“场景变化和即时反馈驱动”。
3. 从“普通画板”转为“声景线描 + 花园生态回应”的共创体验。
4. 从“AI 诊断感”转为“AI 作为温和创作伙伴”。
5. 从“视觉可爱”转为“低刺激、非评判、可回看、适合儿童表达”的完整产品气质。

## 1. 项目定位与边界

### 1.1 产品定位

BreathScape 是一款面向 6–12 岁儿童的情绪表达与轻度调节交互原型。它不以心理诊断、临床治疗或风险评估为目标，而是借鉴艺术疗愈中的非语言表达、象征转化、过程安抚和作品回看机制，让儿童通过绘画、声音和场景变化，把当天模糊、难以语言化的感受转化为可观看、可听见、可保存的“情绪场景日记”。

项目核心不是“让儿童说出自己是什么情绪”，而是帮助儿童完成三个过程：

- 把模糊感受外化为自然元素，例如风、雨、草、花、阳光。
- 通过线描、音效和动画看到自己的表达被温和回应。
- 通过日记卡回看创作过程，为自我觉察和亲子沟通提供入口。

### 1.2 不做什么

本项目必须明确避免以下设计方向：

- 不做心理诊断，不显示“你焦虑”“你抑郁”“你愤怒”等判断。
- 不做量表测评，不输出情绪分数、风险等级、心理画像。
- 不承诺疗效，不写“降低焦虑”“改善心理健康”“治疗情绪问题”。
- 不使用强成瘾机制，不做金币、排行榜、连续签到、任务惩罚。
- 不强迫儿童分享作品，不默认让父母查看全部内容。
- 不把面部识别或语音识别作为核心判断依据。
- 不引入广告、复杂登录、精确定位、社交聊天等高风险模块。

### 1.3 当前 Demo 的合理目标

Demo 阶段只需要证明一个核心机制成立：

儿童可以在一个安全、低压力、非评判的艺术场景中，通过线条、声音和自然元素完成一次情绪表达，并把这个过程保存为可回看的日记。

因此，第一版 Demo 的最小闭环是：

进入页 → 今日心情选择 → AI 场景初始化 → 花园线描画布 → 呼吸收尾 → 今日花园日记卡 → 我的情绪场景日记。

## 2. 儿童 App 风格调研结论与迁移方向

### 2.1 参考产品类型

本方案重点参考以下几类儿童 App，而不是泛泛模仿所有儿童游戏。

| 类型 | 代表产品 | 适合参考的原因 | 对 BreathScape 的迁移方式 |
|---|---|---|---|
| 低刺激开放式玩具 | Pok Pok | 强调开放探索、无输赢、无关卡、低刺激、无语言依赖 | 减少说明型 UI，让儿童通过点、画、听、看变化来理解系统 |
| 小世界探索 | Sago Mini World、Toca Boca World | 强调角色、场景、开放式故事、孩子自由创造 | 把“今日花园”做成可进入的小世界，而不是静态背景 |
| 儿童放松/睡眠/正念 | Moshi Kids、Headspace Kids | 强调安抚音景、呼吸、故事、低刺激体验 | 用柔和声音、慢动画、呼吸光点完成节奏收束 |
| 儿童教育与创意学习 | Khan Academy Kids、Duolingo ABC | 强调短任务、大按钮、角色引导、清晰路径 | 每页只让儿童完成一个动作，避免长文本和复杂选择 |
| 儿童艺术创作 | Crayola Create and Play、Droodle | 强调画画、涂鸦、作品生成、让静态图动起来 | 把儿童线描转化为花园生长、声音反馈和故事日记 |
| 社会情绪学习 | Breathe, Think, Do with Sesame | 强调呼吸、思考、行动、问题解决和情绪词汇 | 呼吸收尾应是场景中的温和动作，而不是医学训练界面 |

### 2.2 儿童 App 共同视觉规律

成熟儿童 App 的界面通常有以下共性：

1. 第一屏有完整的视觉世界，而不是简单背景加按钮。
2. 按钮大、圆、明确，儿童不需要精准点击。
3. 文案短，说明少，动作明确。
4. 反馈来自场景本身，而不是弹窗或系统提示。
5. 色彩明亮但不刺眼，尤其是情绪/放松类产品要避免高刺激。
6. 角色、物件和场景具有生命感，但不一定低幼化。
7. 界面层级少，儿童不会迷路。
8. 没有明显失败状态，不强调“做对/做错”。
9. 主视觉和互动区域优先，功能说明退居后台。
10. 家长入口、设置、隐私和付费信息不能干扰儿童主路径。

### 2.3 对 BreathScape 的关键判断

BreathScape 不应模仿儿童游戏的强奖励和强关卡，也不应模仿成人心理 App 的量表化和数据化。它最适合的风格定位是：

安静但不冷淡，儿童化但不幼稚，艺术化但可交互，AI 存在但不抢戏。

具体来说：

- 模仿 Pok Pok 的低刺激、开放式探索和无输赢机制。
- 模仿 Sago Mini / Toca Boca 的小世界感和物件可互动性。
- 模仿 Moshi Kids 的安抚音景、梦境感与呼吸引导。
- 模仿 Khan Academy Kids 的清晰路径和角色陪伴。
- 模仿 Duolingo ABC 的短任务、大触控目标和即时反馈。
- 避免模仿高刺激儿童游戏的金币、排行榜、强动效、连续打卡和商业弹窗。

## 3. 设计原则

### 3.1 原则一：少解释，多回应

儿童不应通过阅读大量文字来理解产品，而应通过动作和场景反馈理解产品。每个页面只保留必要文案，复杂说明放在汇报文档或教师说明中。

页面文案控制原则：

- 标题不超过 12 个中文字符。
- 主说明不超过 2 行。
- 反馈文案每次只出现 1 句。
- 不出现专业术语，例如“情绪外化”“自我安抚”“生成式反馈”。

### 3.2 原则二：把情绪变成自然隐喻

儿童不需要直接选择“焦虑、愤怒、悲伤”。系统应使用自然隐喻降低表达压力：

- 晴天：轻松一点
- 小雨：有点安静
- 大风：心里有点乱
- 小火苗：有点冲
- 雾蒙蒙：说不太清
- 空土地：没什么力气
- 我也不知道：允许不确定

自然隐喻的作用是把心理标签转化为可创作的场景条件。

### 3.3 原则三：不评价画得好不好

BreathScape 的绘画不是美术训练，也不是图像识别比赛。系统不评价像不像、好不好、对不对，只回应“这一笔给花园带来了什么”。

禁止文案：

- “画得真棒！”
- “识别失败，请重画。”
- “你画错了。”
- “你的情绪改善了。”

推荐文案：

- “风来了，它把花园轻轻带动了。”
- “雨落下来了，土地慢慢喝到水了。”
- “有一点绿色长出来了。”
- “花开了一点点，不用一下子全部打开。”

### 3.4 原则四：AI 是创作伙伴，不是心理专家

Demo 中的 AI 只做三件事：

1. 根据儿童选择的自然隐喻，生成温和的创作建议。
2. 根据绘制元素和线条节奏，触发场景反馈和下一步提示。
3. 根据创作顺序，生成非评判式日记故事。

AI 不能做：

- 诊断儿童状态。
- 预测心理风险。
- 给出临床建议。
- 对家长输出隐性心理画像。
- 用摄像头/麦克风推断真实情绪。

### 3.5 原则五：让场景安放，而不是让情绪消失

BreathScape 的价值不是“消除坏情绪”，而是让情绪被看见、被照料、被安放。因此呼吸收尾不是治疗结论，而是一次节奏收束。

推荐表达：

- “让花园慢慢停下来。”
- “风可以轻一点。”
- “雨可以小一点。”
- “今天的花园先在这里休息。”

不推荐表达：

- “你现在冷静了。”
- “焦虑已经降低。”
- “情绪调节成功。”

## 4. 信息架构与页面流程

### 4.1 页面路径

建议前端路由如下：

```text
/start          进入页
/mood           今日心情选择
/guide          AI 场景初始化
/canvas         声景线描共创画布
/breathe        呼吸收尾
/card           今日花园日记卡
/diary          我的情绪场景日记
/about          项目说明/教师演示说明，可隐藏
```

### 4.2 页面任务分配

| 页面 | 儿童任务 | 系统任务 | 页面关键指标 |
|---|---|---|---|
| 进入页 | 感知这是一个安全花园，点击开始 | 建立产品气质，降低表达压力 | 第一屏有完整世界感，主按钮清晰 |
| 心情选择 | 选择今天更像什么天气/场景 | 记录初始隐喻状态 | 选项大、少字、允许“不知道” |
| AI 引导 | 接受一个温和创作建议 | 把心情转成可画元素 | 不诊断，不贴标签 |
| 画布 | 选择元素并画线 | 识别元素、触发视觉和声音反馈 | 每笔有三重反馈 |
| 呼吸 | 跟随光点进行 2–3 次呼吸 | 降低场景动效，完成收束 | 视觉安静，不像训练工具 |
| 日记卡 | 看见自己的作品和故事 | 生成非评判式回顾 | 不显示分数或疗效 |
| 日记列表 | 回看历史花园 | 本地保存与周回顾 | 像作品展，不像数据报表 |

## 5. 页面详细设计方案

### 5.1 页面一：进入页 /start

#### 页面目标

让儿童一眼理解：这里是一个可以进入、可以画画、不会被评价的安静花园。

#### 布局

iPad 横屏采用大幅插画首页：

- 背景：柔和晨光渐变 + 纸张肌理。
- 中央：一片安静的空白花园，包含小土坡、种子、小灯、风铃、远山、雾气和路径。
- 左上：项目 Logo“息境 BreathScape”。
- 中下：短文案和主按钮。
- 右下：一个很小的“我的日记”入口，可选。

#### 推荐文案

标题：

```text
欢迎来到息境
```

说明：

```text
不用马上说清楚心情。
我们可以先从一条线、一滴雨、一阵风开始。
```

按钮：

```text
开始今日花园
```

#### 视觉要求

- 页面不应像登录页，不要求输入姓名、年龄。
- 不显示“心理疗愈”“检测情绪”“AI 诊断”等词。
- 主按钮高度 68–76px，圆角 32–38px。
- 首页插画完成度必须高于 UI 卡片本身，不能只靠 CSS 几何形状堆叠。

#### 交互

- 点击主按钮后，按钮轻微缩放 0.96 并回弹。
- 种子发出很轻的光点。
- 页面转场采用 500–700ms 淡入，不使用强烈弹跳。

---

### 5.2 页面二：今日心情选择 /mood

#### 页面目标

用自然隐喻替代直接情绪标签，允许儿童说“不知道”。

#### 布局

推荐 7 个卡片，iPad 横屏可使用 4 + 3 布局：

```text
[晴天] [小雨] [大风] [小火苗]
[雾蒙蒙] [空土地] [我也不知道]
```

每张卡片包括：

- 一个大图形或小动画。
- 一个主标签。
- 一句短解释。

#### 选项文案

| 选项 | 标签 | 说明 |
|---|---|---|
| 晴天 | 轻松一点 | 像阳光落在身上 |
| 小雨 | 有点安静 | 像雨落在土地上 |
| 大风 | 心里有点乱 | 像风跑得很快 |
| 小火苗 | 有点冲 | 像小火苗在跳 |
| 雾蒙蒙 | 说不太清 | 像花园里有雾 |
| 空土地 | 没什么力气 | 像土地还在休息 |
| 我也不知道 | 还没想好 | 没关系，可以先画 |

#### 底部提示

```text
没关系，心情不一定马上说得清。我们可以先一起画出来。
```

#### 视觉要求

- 卡片宽度 180–220px，高度 150–180px。
- 每张卡片圆角 28–36px。
- 卡片之间留白充足，避免像考试选项。
- 当前选中卡片轻微放大，显示柔光边框。

#### 交互

- 点击卡片后，卡片中的天气元素微动。
- 允许点击后直接进入 /guide，也可在底部出现“继续”按钮。
- 建议保留“返回花园入口”小按钮，但弱化处理。

### 5.3 页面三：AI 场景初始化 /guide

#### 页面目标

将儿童选择的心情隐喻转化为今日花园的初始氛围和创作建议。

#### 布局

- 左侧 60%：今日花园预览图，根据心情改变天气状态。
- 右侧 40%：AI 引导卡片 + 推荐元素。

#### 场景状态映射

| 心情隐喻 | 场景初始状态 | 推荐元素 | 禁止表达 |
|---|---|---|---|
| 晴天 | 光线柔和、土地明亮 | 花、草、阳光 | “你很开心” |
| 小雨 | 土地湿润、雨线轻落 | 草、花、小灯 | “你很难过” |
| 大风 | 风线明显、草轻摇 | 云、草、小河、阳光 | “你很焦虑” |
| 小火苗 | 暖色偏强、局部跳动 | 雨、草、云 | “你需要冷静” |
| 雾蒙蒙 | 景深变浅、远处模糊 | 小灯、阳光、花 | “你困惑/压抑” |
| 空土地 | 场景空、色彩低、种子静止 | 草、阳光、雨 | “你没能量/低落” |
| 我也不知道 | 中性花园、轻微云雾 | 任意元素 | “系统无法判断” |

#### 文案模板

大风：

```text
今天的花园里风有点急。
它可能需要一些能让它慢下来的东西。
你可以试着画一片云、一条小河，或者几株小草。
```

小雨：

```text
今天的花园有一点湿润。
也许这里正适合种下一颗小种子，
或者点亮一盏小灯。
```

空土地：

```text
今天的花园还很安静。
不用急着让它开满花，
可以先画一株小草，或给它一点阳光。
```

我也不知道：

```text
没关系，花园也可以先不知道。
你可以随便画一条线，
看看它想变成风、雨，还是一点光。
```

#### 交互

- 推荐元素以 2–3 个大图标显示。
- 点击推荐元素后直接进入画布并选中该工具。
- 也提供“我想自己选”按钮。

### 5.4 页面四：场景共创画布 /canvas

#### 页面目标

这是 Demo 的核心页面。儿童在这里通过声景线描与花园共创，系统通过视觉、声音和故事反馈回应每一笔。

#### 布局

iPad 横屏推荐三栏结构：

```text
┌──────────────────────────────────────────────┐
│ 顶部：今日花园 / 返回 / 完成按钮              │
├──────────┬───────────────────────┬───────────┤
│ 工具栏   │       花园画布         │ AI轻提示   │
│ 风雨草花 │  Canvas + 场景底图     │ 元素记录   │
│ 阳光     │                       │ 今日故事   │
└──────────┴───────────────────────┴───────────┘
```

比例建议：

- 左侧工具栏：10%–12%。
- 中央画布：60%–66%。
- 右侧反馈栏：22%–28%。

#### 核心工具

第一版保留 5 个工具：

1. 风
2. 雨
3. 草
4. 花
5. 阳光

不建议第一版加入太多工具，否则会稀释演示重点。

#### 工具按钮规范

| 项目 | 规范 |
|---|---|
| 尺寸 | 72–88px 圆形按钮 |
| 图标 | 32–42px，线性或手绘风 |
| 标签 | 2–3 字，例如“风”“小雨”“小草” |
| 选中态 | 外圈柔光 + 轻微放大 + 背景加亮 |
| 点击反馈 | 0.96 缩放 + 150ms 回弹 |

#### 画布层级

画布不是单一 Canvas，建议分层：

```text
背景插画层：高完成度花园底图
状态动画层：风线、雨滴、光斑、草花生长
儿童线描层：Canvas 记录笔迹
交互热点层：可点击的小灯、种子、风铃等
反馈粒子层：小光点、萤火虫、花粉
```

#### 每笔反馈链路

每次儿童完成一笔后，系统至少触发以下反馈：

1. 画布保留线条。
2. 当前工具对应元素产生动画。
3. 播放轻声音效。
4. 右侧出现一句非评判式反馈。
5. 今日故事素材记录新增一个元素。
6. 场景全局状态轻微变化。

#### 元素反馈规则

| 工具 | 绘制方式 | 视觉反馈 | 声音反馈 | 文案反馈 | 调节含义 |
|---|---|---|---|---|---|
| 风 | 波浪线、长线、旋转线 | 风线流动、花草轻摇 | 轻风声/风铃 | 风来了，它把花园里的声音轻轻带动了。 | 释放、流动 |
| 雨 | 点状线、短竖线、下落线 | 雨滴落下、土地变柔 | 雨滴声 | 雨落下来了，土地慢慢喝到水了。 | 滋养、冲刷 |
| 草 | 向上短线 | 小草生长 | 沙沙声 | 有一点绿色长出来了，它不需要很快。 | 恢复、韧性 |
| 花 | 圆形、弧线、点状花瓣 | 花苞打开、颜色变亮 | 柔和泛音 | 花开了一点点，不用一下子完全打开。 | 打开、被照料 |
| 阳光 | 放射线、光点 | 光线增强、暖色扩散 | 铃音/木琴音 | 阳光落在花园里，刚刚好。 | 支持、能量 |

#### 线条行为识别

第一版不需要复杂机器学习，可用规则近似：

| 线条特征 | 计算方式 | 反馈变化 |
|---|---|---|
| 速度 | strokeLength / duration | 快速风更明显，慢速草生长更柔 |
| 密度 | points count / bounding box area | 密集雨滴更多，花瓣更丰富 |
| 方向 | dx/dy 或角度分布 | 竖线偏雨，横向/曲线偏风 |
| 停顿 | pointermove 间隔 | 停顿后可出现“这里可以先安静一会儿” |
| 长度 | strokeLength | 长线触发大风，短线触发轻风 |

#### 右侧 AI 轻提示

右侧栏不应像聊天窗口。建议设计为“小花园在说”的柔和卡片。

内容包括：

- 当前一句反馈。
- 今日已添加元素：风 ×2、雨 ×1、草 ×3。
- 一个温和建议按钮，例如“加一点阳光”“让花园安静一下”。

示例：

```text
花园回应：
风已经很明显了。
你想不想给它加一片云，让它慢一点？
```

#### 完成按钮

右上角按钮：

```text
让花园慢下来
```

不要写“完成测试”“提交结果”。

### 5.5 页面五：呼吸收尾 /breathe

#### 页面目标

帮助儿童从创作状态过渡到安放状态。呼吸不是医学训练，而是场景节奏收束。

#### 布局

全屏花园背景，中央一个缓慢放大/缩小的光点。

页面元素：

- 中央呼吸光点。
- 底部短文案。
- 右上角“跳过”可选，但不突出。

#### 文案

```text
现在，我们一起让花园慢慢停下来。
```

吸气：

```text
跟着光点吸气
```

呼气：

```text
再慢慢呼气
```

结束：

```text
花园已经安静了一点。
我们把今天留下来吧。
```

#### 动效

- 光点 4 秒放大，4 秒缩小。
- 重复 2–3 次即可。
- 背景风线降低透明度。
- 雨滴减少。
- 花草继续轻轻摇动。
- 背景音乐或环境声逐渐变平稳。

#### 禁止设计

- 不显示“呼吸训练完成”。
- 不显示“心率/压力/焦虑下降”。
- 不显示倒计时压力感过强的数字。

### 5.6 页面六：今日花园日记卡 /card

#### 页面目标

把儿童的创作过程转化为一张可保存、可回看的作品卡。

#### 布局

中心是一张大卡片，包含：

1. 今日花园缩略图。
2. 作品名称输入框，可选。
3. 今日添加元素图标。
4. AI 生成故事回顾。
5. 保存按钮。

#### 推荐文案

标题：

```text
今天的花园日记
```

故事示例：

```text
今天你先给花园带来了一阵风，
后来又画下雨和小草。
风慢慢轻了一些，土地也长出了一点绿色。
这是今天的花园，也是你今天留下的一段心情。
```

另一个示例：

```text
今天的花园一开始很安静。
你先种下了一颗小草，又给它一点阳光。
它没有一下子长大，但已经开始醒来了。
```

#### 命名输入

占位文案：

```text
给今天的花园起个名字吧
```

也允许不命名，默认名称：

```text
6月7日的花园
```

#### 保存按钮

```text
保存到我的花园日记
```

#### 禁止内容

- 不出现“情绪评分”。
- 不出现“改善明显”。
- 不出现“焦虑减少”。
- 不出现“心理状态正常/异常”。

### 5.7 页面七：我的情绪场景日记 /diary

#### 页面目标

让儿童回看作品，让家长或老师看到项目的沉淀价值，但不变成心理监控工具。

#### 布局

- 顶部：我的情绪场景日记。
- 中间：日记卡片网格。
- 右侧或底部：本周小地图。
- 可选：亲子沟通提示。

#### 日记卡片内容

每张卡片包含：

- 缩略图。
- 日期。
- 作品名称。
- 元素小图标，例如风、雨、草。
- 一句故事摘要。

#### 本周回顾文案

```text
这一周，你的花园里出现过风、雨、小草和阳光。
有几天花园很安静，有几天它慢慢热闹起来。
这些都是你留下的心情痕迹。
```

#### 亲子沟通提示

推荐：

```text
你可以问孩子：这阵风后来去了哪里？
```

避免：

```text
你是不是今天不开心？
```

#### 重要边界

父母端未来可以扩展，但第一版不建议做完整父母后台。父母只能看到儿童愿意分享的作品和沟通提示，不应看到心理诊断、风险分数或隐藏画像。

## 6. 视觉设计规范

### 6.1 整体气质

关键词：

```text
低刺激、轻水彩、纸张肌理、自然隐喻、柔和光感、儿童手绘、梦境花园、非评判、安全空间
```

不应出现的气质：

```text
游戏闯关、心理测评、医疗工具、成人冥想 App、数据仪表盘、强商业化儿童游戏
```

### 6.2 色彩系统

| 层级 | 色值 | 用途 |
|---|---|---|
| 晨光米白 | #FFF7E8 | 页面背景、纸张底色 |
| 花影浅紫 | #F1E8FF | 氛围背景、梦境感 |
| 主行动黄 | #F7C948 | 主按钮、阳光、关键行动 |
| 柔橙 | #F2A65A | 按钮渐变、温暖提示 |
| 薄荷绿 | #9ED8C3 | 草、恢复、稳定 |
| 湖蓝 | #8EC5E8 | 雨、水、安静 |
| 花粉 | #F5A8C8 | 花、柔和愉悦 |
| 深文本 | #3F3A4A | 标题正文 |
| 次文本 | #6F687A | 辅助说明 |
| 卡片白 | rgba(255,255,255,0.72) | 半透明卡片 |

### 6.3 色彩使用比例

建议比例：

- 背景低饱和色：60%。
- 花园自然色：25%。
- 主按钮和行动色：10%。
- 状态高亮和粒子：5%。

不要大面积使用高饱和紫色或纯黄色。紫色负责情绪空间和幻想感，黄色负责安全、温暖和行动，二者都应克制。

### 6.4 字体规范

Web 字体栈建议：

```css
font-family: "Nunito", "PingFang SC", "HarmonyOS Sans SC", "Microsoft YaHei", sans-serif;
```

字号建议：

| 类型 | 字号 | 字重 | 行高 |
|---|---:|---:|---:|
| 首页主标题 | 40–48px | 700 | 1.2 |
| 页面标题 | 32–40px | 700 | 1.25 |
| 卡片标题 | 24–28px | 700 | 1.3 |
| 按钮文字 | 22–26px | 700 | 1.2 |
| 正文 | 18–22px | 500 | 1.6 |
| 辅助说明 | 15–17px | 400 | 1.6 |

字体原则：

- 标题要圆润有力。
- 正文要留足行距。
- 不使用 12px 以下小字。
- 不用过度花哨的手写字体承担正文。

### 6.5 插画规范

主插画应具备：

- 轻水彩色块。
- 儿童手绘线条边缘。
- 纸张肌理。
- 柔和低饱和色。
- 场景深度，例如前景、中景、远景。
- 可交互元素预留位置。

花园底图建议包含：

- 中央土地/小坡。
- 一颗小种子。
- 小河或弯曲路径。
- 风铃或小灯。
- 月门/小桥/小屋作为探索点。
- 远山、云、晨雾。
- 留白区域供儿童绘制。

不要让画面过满，否则会压缩儿童绘画空间。

### 6.6 图标规范

图标应采用圆润线性或手绘线条风格。工具图标不应像通用 UI 图标库的冷冰冰线框，而应带有自然元素特征。

建议图标风格：

- 线宽 2.5–3.5px。
- 圆角端点。
- 小面积水彩底色。
- 选中态带柔光边框。

### 6.7 卡片规范

卡片用于心情选择、AI 引导、日记展示。

```css
.card {
  background: rgba(255, 255, 255, 0.72);
  border: 1px solid rgba(255, 255, 255, 0.85);
  border-radius: 32px;
  box-shadow: 0 18px 40px rgba(89, 72, 110, 0.12);
  backdrop-filter: blur(14px);
}
```

但注意：儿童 App 不应过度玻璃拟态。玻璃效果只能轻微使用，不能让文字可读性下降。

## 7. 交互动效规范

### 7.1 动效总原则

动效应缓慢、轻柔、可预测。BreathScape 不适合高频闪烁、强弹跳、快速转场。

建议：

- 页面转场：400–700ms。
- 按钮反馈：120–180ms。
- 花草摇动：2–5s 循环。
- 呼吸光点：4s 吸气 + 4s 呼气。
- 粒子效果：少量、慢速、半透明。

### 7.2 反馈层级

| 层级 | 反馈方式 | 例子 |
|---|---|---|
| 微反馈 | 点击缩放、柔光 | 按钮、工具选中 |
| 局部反馈 | 元素生长、雨滴落下 | 画布笔迹后局部回应 |
| 场景反馈 | 光线变化、风雨强度变化 | 创作过程整体氛围变化 |
| 叙事反馈 | 一句故事文案 | 右侧 AI 轻提示 |
| 收束反馈 | 呼吸光点、声音降低 | 结束阶段 |

### 7.3 禁止动效

- 快速闪烁。
- 过强震动。
- 夸张爆炸粒子。
- 强音效奖励。
- 失败抖动。
- 倒计时催促。

## 8. 声音设计规范

### 8.1 声音定位

声音是 BreathScape 的核心艺术形式之一。声音不是奖励音效，而是“情绪场景被回应”的证据。

### 8.2 声音类型

| 元素 | 声音 | 实现方式 |
|---|---|---|
| 风 | 轻风、风铃 | Web Audio 噪声 + 滤波 + 轻铃音 |
| 雨 | 雨滴 | 短噪声包络随机触发 |
| 草 | 沙沙声 | 高频轻噪声 + 短淡入淡出 |
| 花 | 柔和泛音 | 正弦波/三角波和弦 |
| 阳光 | 木琴/铃音 | 短音阶 + 淡出 |
| 呼吸 | 低频空气感 | 极轻环境音，不抢注意力 |

### 8.3 声音边界

- 默认音量较低。
- 提供静音按钮。
- 不使用尖锐音效。
- 不使用强奖励音。
- 不播放语音诊断。
- 不采集麦克风作为第一版功能。

## 9. AI 规则原型设计

### 9.1 AI 的五个位置

1. 心情隐喻理解：把儿童选择的天气/场景转成花园初始状态。
2. 元素推荐：推荐 2–3 个可以先画的元素。
3. 绘画行为识别：根据线条速度、方向、密度估计反馈强度。
4. 实时生成式反馈：根据元素和场景状态生成一句提示。
5. 日记故事生成：根据创作顺序生成非评判式回顾。

### 9.2 MoodState 数据结构

```ts
type MoodKey =
  | "sunny"
  | "rain"
  | "wind"
  | "flame"
  | "fog"
  | "empty"
  | "unknown";

interface MoodState {
  key: MoodKey;
  label: string;
  metaphor: string;
  sceneTone: {
    brightness: number;
    wind: number;
    rain: number;
    warmth: number;
    fog: number;
    emptiness: number;
  };
  recommendedTools: ToolKey[];
  guideText: string[];
}
```

### 9.3 Stroke 数据结构

```ts
interface StrokeRecord {
  id: string;
  tool: ToolKey;
  points: { x: number; y: number; t: number }[];
  duration: number;
  length: number;
  speed: number;
  density: number;
  direction: "horizontal" | "vertical" | "circular" | "mixed";
  createdAt: number;
}
```

### 9.4 GardenState 数据结构

```ts
interface GardenState {
  mood: MoodKey;
  strokes: StrokeRecord[];
  elements: {
    wind: number;
    rain: number;
    grass: number;
    flower: number;
    sun: number;
  };
  atmosphere: {
    brightness: number;
    softness: number;
    movement: number;
    calmness: number;
  };
  storyFragments: string[];
}
```

### 9.5 实时反馈规则

| 条件 | 系统反馈 |
|---|---|
| 快速画很多风 | “花园里的风已经很明显了。你想不想给它加一片云，让风慢一点？” |
| 连续画很多雨 | “雨已经让土地变得柔软。也许有一株小草准备冒出来了。” |
| 长时间无操作 | “可以先留一块安静的地方。也可以轻轻点一下土地，看看种子想不想醒来。” |
| 画面很空 | “这片花园还有很多空白。你可以让它继续安静，也可以给它加一点声音。” |
| 先风后草 | “风经过以后，有一点绿色慢慢站起来了。” |
| 先雨后花 | “雨落下以后，花苞好像准备打开一点。” |
| 先空土地后阳光 | “一点光落下来，土地开始醒了一点。” |

### 9.6 禁用词与替代表达

| 禁用词 | 替代表达 |
|---|---|
| 焦虑 | 风有点急 |
| 抑郁 | 花园有点安静 |
| 愤怒 | 小火苗跳得有点快 |
| 治疗 | 陪你画出来 |
| 改善 | 慢慢安放 |
| 冷静下来 | 让花园慢一点 |
| 情绪异常 | 今天的花园有自己的样子 |
| 评分 | 留下的痕迹 |

### 10.5 Canvas 绘制实现

核心要求：

- 支持鼠标、手指、触控笔。


## 14. 验收标准

### 14.1 功能验收

- 用户能从首页完成完整流程并保存日记。
- 7 个心情选项都能进入不同引导文案。
- 5 个工具均可绘制并触发反馈。
- 每笔绘制后至少出现视觉、声音、文案三类反馈。
- 呼吸页能完成 2–3 次节奏收束。
- 日记页能展示保存结果。
- 刷新页面后日记仍存在。

### 14.2 视觉验收

- iPad 横屏 1024×768 下无内容重叠。
- 主画布占页面主视觉，不被工具栏挤压。
- 所有按钮触控面积不小于 44px，建议主按钮不小于 64px 高。
- 页面整体低刺激，无高频闪烁和强对比压迫。
- 视觉风格统一：轻水彩、纸张肌理、圆润卡片、自然元素。

### 14.3 文案验收

- 不出现心理诊断词。
- 不出现疗效承诺。
- 不出现评分或风险等级。
- 不出现失败、错误、重画等评价性语言。
- 每页文案短、清楚、适合儿童理解。

### 14.4 安全与隐私验收

- 不采集真实姓名、精确位置、照片、视频、麦克风数据。
- 不接入第三方广告。
- 不接入儿童画像分析。
- 不默认上传作品。
- 所有保存仅在本地 localStorage。

## 15. 评审风险与修正策略

### 风险一：被认为“艺术疗愈”概念过度

如果汇报中宣称“治疗儿童情绪问题”，评审会质疑证据不足。正确策略是把项目定位为“儿童情绪表达与轻度调节原型”，强调非诊断、非临床、非疗效承诺。

### 风险二：被认为只是画画 App

如果画布只有绘画功能，没有情绪隐喻、声景反馈和日记回看，项目会降级为普通儿童画板。必须突出三点：

- 心情隐喻进入。
- 每笔触发场景和声音回应。
- 创作过程生成非评判式日记。

### 风险三：AI 存在感不足

如果 AI 只写在方案里，页面上没有体现，评审会认为 AI 是包装。解决方式：

- /guide 页面明确体现“AI 把心情转成创作建议”。
- /canvas 页面实时根据绘制顺序更新反馈。
- /card 页面根据元素顺序生成故事。

### 风险四：儿童产品安全边界不足

如果出现面部识别、家长监控、情绪评分、云端上传等设计，会引发隐私和伦理风险。第一版应坚持本地保存、无诊断、无广告、无社交。

### 风险五：视觉像临时原型

如果花园只由 CSS 圆形、渐变和简单图标组成，难以达到 App Store 截图级完成度。必须生成或绘制一张高完成度场景底图，再用代码叠加交互层。

## 16. 给开发工具 / Codex 的执行提示词

```text
请基于当前 BreathScape Demo 项目重构为 iPad 横屏优先的儿童情绪场景日记 Web App。项目定位是 6–12 岁儿童的非诊断式情绪表达与轻度调节原型，不做心理诊断、不做量表、不显示情绪评分、不做疗效承诺。

核心链路为：/start 进入页 → /mood 今日心情选择 → /guide AI 场景初始化 → /canvas 声景线描共创画布 → /breathe 呼吸收尾 → /card 今日花园日记卡 → /diary 我的情绪场景日记。

视觉风格要求：低刺激、轻水彩、纸张肌理、晨光黄 + 花影紫主色，辅以薄荷绿、湖蓝、花粉；圆润大按钮；iPad 横屏 1024×768 必须完整可用。首页和画布要像一个可进入的情绪花园小世界，而不是普通表单页面。

功能要求：
1. /mood 提供 7 个心情隐喻卡片：晴天、小雨、大风、小火苗、雾蒙蒙、空土地、我也不知道。
2. /guide 根据选择生成不同花园氛围和 2–3 个推荐绘制元素，文案不能出现焦虑、抑郁、愤怒、治疗、改善等诊断或疗效词。
3. /canvas 提供 5 个工具：风、雨、草、花、阳光。支持 Pointer Events 绘制。每次 pointerup 后分析线条速度、长度、方向、密度，并触发视觉反馈、Web Audio 声音反馈和一句温和文案。
4. 每个工具必须有不同反馈：风线流动、雨滴落下、小草生长、花朵开放、阳光光斑。
5. /breathe 显示缓慢放大缩小的呼吸光点，引导 2–3 次呼吸，同时让风雨动效减弱。
6. /card 根据元素绘制顺序生成今日花园故事卡，可命名并保存到 localStorage。
7. /diary 展示保存的日记卡和本周小地图。
8. 全项目不接入广告、不接入登录、不上传儿童数据。

请优先保证完整可演示闭环、iPad 横屏适配、儿童化视觉完成度和非评判式文案边界。不要把界面做成心理测评、成人冥想工具或普通画板。
```

## 17. 最终判断

这一版 BreathScape 的设计成败不取决于“是否足够可爱”，而取决于是否真正建立了一个证据链：

儿童难以直接语言化的情绪 → 自然隐喻选择 → 可绘制元素 → 场景和声音即时回应 → 呼吸收束 → 非评判式日记回看。

如果这个链条成立，项目就不只是儿童画画 App，也不只是 AI 包装的心理页面，而是一个有明确设计研究价值的儿童情绪表达交互原型。反之，如果页面只停留在可爱插画和按钮堆叠，或者过度宣称心理疗效，都会成为评审攻击点。

本版本的核心设计标准应坚持一句话：

让孩子不用先说清楚，也能把今天的心情画出来、听见它、看见它，并把它温和地留下来。
